# Background

The current install method for tinyspec requires the user to have the Rust toolchain (Cargo) installed (`cargo install tinyspec`). This is a barrier for users who just want the CLI binary without setting up a Rust development environment.

Release artifacts already exist on GitHub for macOS (arm64, x86_64), Linux (x86_64), and Windows (x86_64), but no SHA256 checksums are published alongside them. The marketing site (Next.js app in `site/`) and getting-started docs only document the Cargo install method.

# Proposal

Create an `install.sh` script that auto-detects the user's OS and architecture, downloads the correct pre-built binary from GitHub releases, verifies its SHA256 checksum, and installs it to `~/.local/bin`. The script should support macOS (arm64 + x86_64), Linux (x86_64), and Windows via WSL.

Host `install.sh` as a static file on the marketing site (`site/public/install.sh` â†’ `https://tinyspec.dev/install.sh`) so users can install with a single curl command.

Update the release workflow to generate and publish SHA256 checksums. Update the marketing site Install component and getting-started docs to present both install methods: the quick curl one-liner and the existing Cargo install.
