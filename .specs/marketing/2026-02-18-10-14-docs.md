# Background

The tinyspec site is a Next.js static site with a single landing page covering Hero, Features, Install, Header, and Footer sections. Beyond a 5-step quickstart, there is no documentation. Users who want to learn about spec format, CLI commands, configuration, templates, multi-repo support, the dashboard, or Claude Code integration have no reference material.

# Proposal

Add a full `/docs` section to the site using MDX files with a shared layout. The docs section will have:

- A persistent sidebar navigation (collapsible hamburger on mobile)
- Clean long-form typography consistent with the existing design system
- MDX-based authoring for easy maintenance and extensibility

**Doc pages:**

1. **Getting Started** — install, init, create a first spec, end-to-end walkthrough
1. **Concepts** — spec structure (front matter, Background, Proposal, Implementation Plan, Test Plan), task groups & subtasks, spec lifecycle
1. **CLI Reference** — all commands (init, new, list, view, check, uncheck, format, dashboard) with usage and options
1. **Claude Code Integration** — the three skills (/tinyspec-refine, /tinyspec-work, /tinyspec-task), how they work, CLAUDE.md setup
1. **Configuration** — .tinyspec.toml, multi-repo support, custom templates, template variables
1. **Dashboard** — TUI dashboard usage and features
