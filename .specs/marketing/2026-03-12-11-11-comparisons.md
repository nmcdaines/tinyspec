# Background

There are a series of alternative solutions to tinyspec each with their own tradeoffs. This proposal is designed to provide detailed comparisons of the competing solutions as an easy reference for humans and llms.

# Proposal

Create a comparisons section under `/docs/comparisons/` on the marketing site. This includes:

- An **overview page** at `/docs/comparisons` with a feature matrix table comparing tinyspec against all alternatives across key dimensions: spec format/structure, CLI tooling, workflow integration, and formatting capabilities.
- **Individual detail pages** for each alternative (e.g., `/docs/comparisons/openspec`) with prose comparisons covering strengths, weaknesses, and key differentiators.
- Follow the existing site pattern: `page.tsx` + `content.mdx` per page, wired into `nav.ts`.

Alternatives to compare:

- OpenSpec (https://github.com/Fission-AI/OpenSpec)
- GitHub SpecKit (https://github.com/github/spec-kit)
- BMad (https://github.com/bmad-code-org/BMAD-METHOD)
- GSD (https://github.com/gsd-build/get-shit-done)
