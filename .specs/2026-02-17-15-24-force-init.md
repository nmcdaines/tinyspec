# Background

When tinyspec is updated, the skill prompts in `src/spec.rs` may change, but `tinyspec init` skips writing command files that already exist. There is no way to replace them with the latest version short of manually deleting and re-running init.

# Proposal

Add a `--force` / `-f` flag to `tinyspec init` that overwrites existing command files with the latest skill prompts.

# Implementation Plan

- [x] A: Add `--force` flag to CLI
  - [x] A.1: Add `--force` / `-f` flag to the `Init` variant in `Commands` enum in `src/main.rs`
  - [x] A.2: Pass the `force` parameter through to `spec::init()`
- [x] B: Update init logic in `src/spec.rs`
  - [x] B.1: Update `init()` signature to accept `force: bool`
  - [x] B.2: When `force` is true, skip the `path.exists()` check and always write the file
