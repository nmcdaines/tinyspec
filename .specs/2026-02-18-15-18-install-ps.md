# Background

We have an install script (`install.sh`) for Linux and macOS users that provides a one-line install experience. Windows users currently have no equivalent â€” the install.sh script errors on Windows and directs them to `cargo install tinyspec`. However, the Windows binary (`x86_64-pc-windows-msvc.zip`) is already built and published in GitHub releases with SHA256 checksums.

# Proposal

Create an `install.ps1` PowerShell script that mirrors `install.sh` behavior: detect architecture, download the correct binary from GitHub releases, verify SHA256 checksum, install to `$env:LOCALAPPDATA\tinyspec`, and auto-add to the user's PATH. Serve it at `tinyspec.dev/install.ps1` so users can run `irm https://tinyspec.dev/install.ps1 | iex`. Update the marketing site and docs to auto-detect the user's OS and show the appropriate install command.
